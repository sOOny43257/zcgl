# 数据库连接配置说明

## 配置文件位置

数据库连接配置位于以下两个文件中：

### 1. 环境变量：`.env`（项目根目录）

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=zcgl
DB_USERNAME=root
DB_PASSWORD=
```

### 2. 连接详细配置：`config/database.php`

`connections.mysql` 数组中的配置项会从 `.env` 读取：

```php
'mysql' => [
    'driver' => 'mysql',
    'host' => env('DB_HOST', '127.0.0.1'),
    'port' => env('DB_PORT', '3306'),
    'database' => env('DB_DATABASE', 'zcgl'),
    'username' => env('DB_USERNAME', 'root'),
    'password' => env('DB_PASSWORD', ''),
    'unix_socket' => env('DB_SOCKET', ''),
    'charset' => 'utf8mb4',
    'collation' => 'utf8mb4_unicode_ci',
    'prefix' => '',
]
```

## 修改数据库配置

1. 编辑 `.env` 文件，修改 `DB_*` 相关参数
2. 运行 `php artisan config:clear` 清除配置缓存

## MySQL 客户端工具路径

备份和还原功能需要 `mysqldump` 和 `mysql` 命令。系统会按以下顺序自动查找：

1. `.env` 中 `MYSQL_BIN_DIR` 指定的目录
2. 系统 PATH 中的 `which mysqldump` / `which mysql`
3. 常见安装路径（XAMPP、Homebrew、Linux 标准路径等）

如果自动探测失败，在 `.env` 中手动配置：

```env
# XAMPP
MYSQL_BIN_DIR=/Applications/XAMPP/xamppfiles/bin

# Homebrew
MYSQL_BIN_DIR=/opt/homebrew/bin

# Linux 标准
MYSQL_BIN_DIR=/usr/bin

# Docker 容器内通常无需配置（已在 PATH 中）
MYSQL_BIN_DIR=
```

修改后运行 `php artisan config:clear` 生效。

## 命令行连接

```bash
# 使用 XAMPP 自带的 MySQL 客户端
/Applications/XAMPP/xamppfiles/bin/mysql -u root zcgl

# 或使用 PHP artisan 进入数据库
php artisan db
```
